Phase 8.1 Card Semantics + Sensitive Action Semantics:
- Fixed decision title generation so MVP-focus decisions and pricing decisions get distinct, content-specific titles.
- MVP focus card title is now "Focus MVP on Memory Search, Bootstrap Review, and Smart Reminders".
- Pricing card title is now "Pricing Strategy: 29-49 USD" and body retains the 29-49 USD/month detail.
- Sensitive item review cards now use a privacy-aware action matrix: Keep Restricted / Mark Reviewed / Archive / View Source.
- keep_restricted does not create a normal Active MemoryItem and writes InteractionSignal interpreted_signal "kept_sensitive_item_restricted".
- mark_reviewed does not create a normal Active MemoryItem and writes InteractionSignal interpreted_signal "marked_sensitive_item_reviewed".
- Normal card Save still promotes to Active MemoryItem; Archive still does not create MemoryItem.
- Direct promote_to_active_memory calls against sensitive review cards are blocked with VALIDATION_ERROR.
- Review UI renders all backend-provided BootstrapReviewCard actions rather than hardcoding card action labels.
- No duplicate card builders or frontend-only title hacks were introduced.
- Small type-level extension added MemoryCandidate statuses "restricted" and "reviewed"; external JSON schemas remain unchanged.
- Repo-health verdict: improved.

Phase 9 Persistence Hardening:
- Introduced LifeOSStorage repository boundary with EventRepository, BootstrapReviewRepository, BootstrapReviewCardRepository, MemoryCandidateRepository, MemoryItemRepository, InteractionSignalRepository, and FocusStateRepository.
- Added SQLite-backed implementation under src/server/storage/sqlite.
- Default runtime storage is LIFEOS_STORAGE_DRIVER=sqlite with LIFEOS_SQLITE_PATH=.lifeos/lifeos.dev.sqlite.
- In-memory storage remains available as an explicit test/dev adapter and is no longer the default runtime path.
- SQLite initialization is idempotent and creates events, bootstrap_reviews, bootstrap_review_cards, memory_candidates, memory_items, interaction_signals, and focus_states.
- Canonical JSON payloads are stored as text blobs while preserving typed repository access.
- Domain behavior and external API response shapes are preserved.
- Rollback path: set LIFEOS_STORAGE_DRIVER=memory to run the previous nonpersistent adapter, or revert the Phase 9 commit if SQLite must be removed.

Phase 10 Local Ingestion Gateway Hardening:
- Added src/server/localIngestionGateway.ts as the local intake boundary for explicit approved text sources.
- Added POST /api/v1/ingestion/preview.
- Preview normalizes text, computes sha256 checksums, reports excerpts/line counts/character counts, and dedupes identical source payloads.
- Bootstrap Review now uses the gateway normalization path before creating Events.
- Events retain source refs and content checksums.
- Broad OS file scanning remains out of scope.
- Repo-health verdict: improved.

Phase 11 Smarter Review Queue State:
- Added src/server/reviewQueue.ts to decorate BootstrapReviewCards with backend-owned queue metadata.
- Added GET /api/v1/review-queue?bootstrap_id=...
- Queue response reports pending/saved/archived/restricted/reviewed/rejected state counts.
- Saved, archived, restricted, and reviewed states are derived from canonical card status and linked sensitive handling metadata.
- Focus Mode suppression still applies before returning visible cards.
- Repo-health verdict: improved.

Phase 12 UI Polish / Source Drawer:

Phase 13 Semantic Memory Search + Source-grounded Answers:
- Added src/server/semanticMemory.ts for deterministic local semantic/lexical retrieval over active MemoryItems.
- GET /api/v1/memory/search now returns scored results with matched terms and source Event excerpts.
- Added POST /api/v1/memory/answer for source-grounded answers with citations.
- Answers are generated only from saved active memories and source Events; no external model is called.
- If retrieval has no sufficient source support, the answer is marked grounded=false with no citations.
- Review UI now includes a Memory Search panel that renders backend-provided grounded answers and citations.
- Repo-health verdict: improved.

Known constraints:
- Storage is SQLite-backed local persistence for runtime, with canonical JSON text blobs rather than normalized relational columns.
- Node's built-in node:sqlite module is used to avoid adding native npm dependencies; it may emit experimental runtime warnings on some Node versions.
- Local source scanning is limited to explicit user-provided text/file payloads for this first slice.
- External actions are not executed automatically.
- UI is a local review surface, not a full tray app or dashboard.
- Broad OS file scanning, production auth, semantic search, real calendar/email integrations, and external action execution remain non-goals for this demo slice.
